SS_Game.h

SS_Game

SS_Game represents the base application class. Your game should subclass SS_Game overriding the Run method. The Run method will contain your setup code and main loop.

You must then edit the main.cpp file, replacing all instances of Demo_Game with the name of your own SS_Game subclass.

Member Data

boolbQuitFlag that tells the game to quit.
Uint16worldCountThe number of worlds in the world stack.
SS_World**worldsThe world stack.
SS_World*worldThe active world.

Constructors

SS_Game()

When the base game class is instantiated it creates a screen or window based on your settings in the SS_Config.h file, initializes the audio system, and generates sine and cosine arrays for use in game calculations. If your game needs to maintain its own game-wide data it should override the Init method and do its allocation and initialization there.
