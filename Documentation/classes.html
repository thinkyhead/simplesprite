Class Reference

SimpleSprite has a lot of classes and methods - many more than you will ever use in most games. This documentation is pretty thorough and contains many examples. Of course the best way to learn SimpleSprite is to examine the sample code. The samples are small and simple and each showcases a different concept.

There are a lot more class methods than you will ever need to use in most games so don't feel too overwhelmed.

I've followed certain conventions in designing these classes. Many of these classes have a Copy Constructor, an Init method, an operator= override, and a Clone method.
